# Руководство по внедрению дополнительных подсистем

## 1. Вводная часть

### 1.1. Цель документа

Настоящее руководство предназначено для администраторов и ответственных специалистов заказчика, выполняющих первоначальную настройку расширенной функциональности в существующей информационной базе «Управление IT-фирмой».

Внедряемый функционал охватывает следующие направления:
- автоматизация циклов обработки клиентских заявок на обслуживание;
- организация уведомительного механизма через Telegram-каналы;
- автоматизированный расчёт управленческой заработной платы;
- формирование управленческой отчётности по труду и финансам.

### 1.2. Обязательные условия перед началом работ

- Все манипуляции выполняются в продуктивной базе, содержащей актуальные учётные данные.
- Перед любыми действиями создайте полную резервную копию информационной базы.
- Большинство операций требуют учётной записи с административными привилегиями.

## 2. Развёртывание обновлённой конфигурации

### 2.1. Загрузка конфигурации

Обновление поставляется в виде файла дампа информационной базы.

**Порядок действий:**

1. Получите файл с расширением `.dt` и сохраните его на локальный компьютер.
2. Запустите Конфигуратор с правами администратора.
3. Выберите пункт меню «Администрирование» → «Загрузить информационную базу».
4. Укажите путь к полученному файлу `.dt`.
5. Подтвердите загрузку и дождитесь завершения процесса импорта данных.
6. После загрузки закройте Конфигуратор и откройте его заново.

> **Важно:** При загрузке вся текущая конфигурация будет заменена. Убедитесь, что в базе нет уникальных доработок, которые потребуется перенести вручную.

### 2.2. Адаптация параметров запуска

Для корректной инициализации новых объектов метаданных:

1. Откройте параметры запуска 1С:Предприятия в Конфигураторе.
2. В поле дополнительных параметров укажите ключ `ЗапуститьОбновлениеИнформационнойБазы`.
3. Запустите базу в пользовательском режиме (F5).
4. При появлении запроса на реорганизацию данных подтвердите действие.
5. После завершения обновления удалите указанный параметр из настроек запуска.

## 3. Организация оповещений через Telegram

### 3.1. Регистрация служебного бота

1. Обратитесь к официальному боту-регистратору в Telegram — @BotFather.
2. Инициируйте диалог через команду `/start`.
3. Отправьте запрос на создание нового бота: `/newbot`.
4. Присвойте боту отображаемое имя и уникальный идентификатор, оканчивающийся на `bot`.
5. Сохраните полученный токен — он потребуется в дальнейшем.

### 3.2. Подготовка канала для уведомлений

1. Организуйте групповой чат для приёма уведомлений.
2. Включите созданного бота в состав участников группы.
3. Делегируйте боту административные полномочия в чате.
4. Отправьте тестовое сообщение для активации взаимодействия.
5. Извлеките из ответа идентификатор чата (`chat.id`) и зафиксируйте его.

## 4. Конфигурирование справочной информации

### 4.1. Ведение номенклатурного состава

В справочнике «Номенклатура» должны присутствовать позиции для отражения услуг по абонентскому обслуживанию:

| Позиция | Характер | Использование |
|---------|----------|---------------|
| Абонентская плата | Услуга | Ежемесячные начисления по договорам |
| Работы специалиста | Услуга | Фиксация выполненных работ |

### 4.2. Кадровое наполнение системы

Для каждого трудоустроенного сотрудника создайте запись в справочнике «Физические лица»:

- определите категорию: «Специалист» или «Административный персонал»;
- отмечайте уволенных сотрудников соответствующим флагом;
- при наличии полномочий свяжите учётную запись пользователя с карточкой сотрудника.

### 4.3. Фиксация условий оплаты

В регистре «Условия оплаты сотрудников» установите для каждого работника:

- размер месячного оклада;
- для специалистов — долю от стоимости реализованных работ (допускается нулевое значение).

Отсутствие установленной доли у специалиста препятствует проведению документов обслуживания.

### 4.4. Формирование календарей работы

Для корректного расчёта оплаты труда потребуется настроить графики:

1. Используйте встроенные шаблоны («Пятидневка», «Шестидневка») или разработайте собственные.
2. Заполните регистр «Графики работы» через специализированную обработку.
3. Укажите периоды действия и выходные дни.
4. Дополнительно внесите корректировки на праздничные и перенесённые даты.

## 5. Установка системных констант

Все управляющие параметры концентрируются в разделe «Администрирование».

### 5.1. Каналы связи с Telegram

| Параметр | Источник данных |
|----------|-----------------|
| Ключ доступа бота | Токен, выданный @BotFather |
| Идентификатор чата | Получен через API-запрос |
| Интервал ожидания ответа | Указывается в секундах (0 — без ограничения) |

Для верификации настроек применяйте обработку «Тестирование Telegram-подключения» из подсистемы обслуживания.

### 5.2. Учётные параметры абонентского обслуживания

| Параметр | Действие |
|----------|----------|
| Номенклатура для абонплаты | Выберите созданную позицию |
| Номенклатура для работ специалиста | Выберите соответствующую позицию |

### 5.3. Базовые параметры вычислений

| Константа | Рекомендуемое значение | Назначение |
|-----------|------------------------|------------|
| Ставка НДФЛ | 13% | Расчёт налога с доходов физлиц |
| Лимит дней отпуска | 28 | Порог срабатывания предупреждения |

## 6. Распределение полномочий между пользователями

Новый функционал требует назначения четырёх дополнительных профилей доступа в зависимости от должностных обязанностей.

### 6.1. Ролевая модель

| Роль | Целевая аудитория | Функциональные возможности |
|------|-------------------|---------------------------|
| Специалист | Технические специалисты | Ведение заявок, календарь, отчёт по выработке |
| Менеджер | Руководители среднего звена | Полный цикл управления заявками, контрагенты |
| БухгалтерИТФирмы | Бухгалтеры | Акты, массовые реализации, фин. аналитика |
| КадровикРасчетчик | HR-специалисты | Расчёт зарплаты, управление отпусками |

### 6.2. Процесс наделения правами

1. Перейдите в раздел «Пользователи» через панель администрирования.
2. Откройте карточку требуемого сотрудника.
3. На вкладке доступа добавьте соответствующий профиль.
4. Подтвердите изменения и попросите пользователя перезапустить сеанс.

## 7. Внедрение зарплатного модуля

### 7.1. Предварительные условия

Зарплатный блок становится активным после выполнения всех шагов из раздела 4. Дополнительно убедитесь в корректности:

- заполненных процентов для специалистов;
- актуальности графиков работы;
- установленной ставки НДФЛ.

### 7.2. Функционал расчётных документов

Доступны следующие виды расчётных операций:

- **Базовая оплата труда** — начисление окладов и производственных процентов.
- **Премиальные выплаты** — разовые поощрения сотрудников.
- **Погашение задолженности** — отражение фактических перечислений.

Документы обслуживания клиентов автоматически формируют проводки по регистру выполненных работ, которые участвуют в расчёте зарплаты.

### 7.3. Организация отпускного процесса

Документ «График отпусков» обеспечивает:

- годовое планирование отдыха сотрудников;
- визуальный контроль загрузки через диаграмму Ганта;
- контроль соблюдения установленного лимита.

## 8. Валидация основных сценариев

После завершения конфигурации протестируйте критические бизнес-процессы.

### 8.1. Обработка клиентской заявки

1. Войдите в систему как координатор.
2. Оформите новый документ обслуживания, заполнив все обязательные поля.
3. Сохраните документ — контролируйте поступление уведомления в Telegram.
4. Попробуйте провести документ — система должна верифицировать период действия договора.

### 8.2. Формирование акта реализации

1. Авторизуйтесь как финансист.
2. Создайте «Реализацию товаров и услуг» с выбором абонентского договора.
3. Активируйте автоматическое заполнение — должны подтянуться строки абонплаты и работ.
4. Проверьте макет печатной формы акта.

### 8.3. Автоматизированное формирование пакета актов

Используйте обработку «Массовое создание актов» в начале месяца:

- задайте отчётный интервал;
- дождитесь построения списка активных договоров;
- инициируйте фоновое формирование документов.

### 8.4. Расчёт заработной платы

1. Введите документ начисления за отчётный период.
2. Проверьте корректность вычислений по окладам и процентам.
3. Сформируйте выплатной документ с автозаполнением сальдо.

## 9. Отчётная аналитика

Система предоставляет четыре ключевых отчёта для управленческого контроля:

| Отчёт | Адресат | Содержание |
|-------|---------|------------|
| Выработка специалистовв | Исполнители, кадровики | Часы и суммы начислений по специалистам |
| Анализ выставленных актов | Финансисты | Объёмы начислений по абонентским договорам |
| Расчёты с сотрудниками | Кадровики | Детализация начислений, выплат и остатков |
| Расход запланированных отпусков | Кадровики | Сопоставление плановых и фактических отпусков |

Отчёты строятся исключительно на данных регистров, что обеспечивает высокую скорость и достоверность.
